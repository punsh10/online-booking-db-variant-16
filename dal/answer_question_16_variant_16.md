Тестирование репозиториев без реальной базы данных необходимо, чтобы тесты работали быстро, не зависели от внешнего сервера и не портили настоящие данные. Существует три основных подхода.

1.SQLite в памяти. Вместо MySQL тест подключается к SQLite с параметром :memory:. База создаётся в оперативной памяти и автоматически исчезает после теста. В PDO достаточно изменить строку подключения: new PDO('sqlite::memory:'). Затем выполняется schema.sql для создания таблиц. Плюс — не нужен реальный сервер. Минус — SQLite не поддерживает ENUM и некоторые другие фичи MySQL.

2.Транзакции с откатом. Тест подключается к реальной базе, но в начале вызывает beginTransaction(), а в конце rollBack(). Все изменения, сделанные тестом, отменяются. База остаётся в исходном состоянии. Плюс — тестируем на настоящей MySQL. Минус — нужен доступ к серверу, тесты медленнее.

3.Моки (заглушки). Реальный репозиторий заменяется поддельным объектом через createMock(). Мок возвращает заранее заданные данные без обращения к базе. Плюс — тесты работают мгновенно. Минус — мы не тестируем настоящие SQL-запросы, только логику вызывающего кода.
